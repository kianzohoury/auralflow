class auralflow.trainer.CallbackManager[source]#

Bases: object

Handles a group of training callbacks.

By default, calling a method corresponding to an event will have no effect. Each event can be handled by either implementing the corresponding handler method manually, or by calling add_callback(...), which bins a TrainingCallback to its event.

Note that this class currently has limited functionality and is not stable due to the current implementation of trainer.ModelTrainer.

Examples

>>> from auralflow.trainer import CallbackManager, LayersVisualCallback
>>> callbacks = CallbackManager()
>>> # create a new callback
>>> layers_callback = LayersVisualCallback(...)
>>> # add to callbacks manager
>>> callbacks.add_callback(layers_callback)

Wrapper class that trains a SeparationModel model.

All subclasses must implement the full_forward and scheduler_step methods.

Parameters:

  • criterion (Union[nn.Module, Callable[..., Tensor]]) -- Loss criterion.

  • optimizer (Optimizer) -- Optimizer.

  • scheduler (object) -- LR scheduler. See torch.optim.lr_scheduler.

  • use_mixed_precision (bool) -- If True, enables automatic mixed precision. Default: True.

  • grad_scaler (optional[GradScaler]) -- Gradient scaler. Default: None.

  • max_grad_norm (optional[float]) -- Maximum gradient norm used for gradient clipping. Gradient clipping is disabled if None is passed in. Default: 100.0.

  • stop_patience (int) -- Number of epochs to train before stopping if the validation loss does not improve. Default: 5.

  • min_improvement (float) -- Minimum improvement in the validation loss required to reset the stop patience counter. Default: 0.01.

  • max_lr_reductions (int) -- Maximum number of learning rate reductions if self.scheduler is an instance of ReduceLROnPlateau. Default: 5.

  • checkpoint_dir (optional[str]) -- Directory to save or load object states from. No checkpoints will be saved if a path is not specified. Default: checkpoints.

  • overwrite_checkpoint (bool) -- If True, overwrites previous checkpoints. Default: True.

  • best_checkpoint (bool) -- If True, saves the best model checkpoint as best.pth. Default: True.

  • silent_checkpoint (bool) -- If True, suppresses checkpoint logging. Default: True.

  • resume_training (bool) -- If True, resumes training from checkpoint files. Otherwise, training is restarted. Default: True.

  • use_tensorboard (bool) -- If True, enables tensorboard. Default: True.

  • write_iter_loss (bool) -- If True, logs iteration training loss if tensorboard is enabled. Default: True.

  • write_epoch_loss (bool) -- If True, logs epoch training and validation loss if tensorboard is enabled. Default: True.

  • visualize_weights (bool) -- If True, logs model weights by layer if tensorboard is enabled. Default: True.

  • visualize_gradients (bool) -- If True, logs gradients with respect to layers if tensorboard is enabled. Default: True.

  • visualize_norm (bool) -- If True, logs the 2-norm of each weight/gradient if tensorboard is enabled. Default: True.

  • visualize_waveform (bool) -- If True, logs target source estimates as waveform images if tensorboard is enabled. Default: True.

  • visualize_spectrogram (bool) -- If True, logs target source estimates as spectrogram images if model is an instance of a SpectrogramMaskModel and tensorboard is enabled. Default: True.

  • play_estimate (bool) -- If True, logs listenable target source estimates if tensorboard is enabled. Default: True.

  • play_residual (bool) -- If True, logs listenable background track estimates if tensorboard is enabled. Default: True.

  • image_save_freq (int) -- Frequency in epochs at which to save images. Default: 5.

Runs the forward pass for the model and calculates the loss.

Gets called within the main training loop, via run_training(...) and run_validation(...).

All subclasses must implement this method.

Parameters:

  • target (Tensor) -- Target audio of dimension (batch, channels, time)

Returns:

Tensor -- The loss.

Examples:

# >>> from auralflow.trainer import ModelTrainer # >>> # basic implementation for training spectrogram mask models # >>> class MyCustomTrainer(ModelTrainer): # ... # ... def __init__(self, *args, **kwargs): # ... super(MyCustomTrainer, self).__init__(*args, **kwargs) # ... # ... def full_forward(self, mixture, target): # ... # data transformations # ... mix_spec, mix_phase = self.model.to_spectrogram(mixture) # ... target_spec, _ = self.model.to_spectrogram(target) # ... # model output # ... estimate_spec, data = self.model.forward(mix_spec) # ... # calculate loss # ... loss = self.model.criterion(estimate_spec, target_spec) # ... return loss

class auralflow.trainer.TrainingCallback(event: str)[source]#

Bases: ABC

Abstract base class for all training callbacks.

This class should not be instantiated, but rather subclassed.

Parameters:

event (str) -- Event that triggers the callback: 'on_iteration_end' | 'on_loss_end' | 'on_epoch_start' | 'on_epoch_end'.

Examples

>>> from auralflow.trainer import TrainingCallback
>>> class MyCustomCallback(TrainingCallback):
...     def __init__(self, event, ...):
...         super(MyCustomCallback, self).__init__(event)
...         # handle instance attributes
...         ...
...     def __call__(self, **kwargs):
...         # define callback here
...         ...
>>> my_callback = MyCustomCallback(event="on_epoch_end", ...)
>>> my_callback(...)
